# amista 残タスク一覧(唯一の正)

## このファイルの扱い方(ルール)

- このファイルが残タスク一覧の唯一の正(Single Source of Truth)です。Claude Code・Cowork(Claude)のどちらも、残タスクを確認・報告する際は必ずこのファイルをその場で読み直してから表示してください。記憶・チャット履歴・memory.mdからの再構成は行わないこと。
- タスクの状態が変わった場合(完了・新規発見・進捗変化)は、その場でこのファイルを直接編集し、git commit・pushまで行うこと。編集を後回しにしない。
- 番号は既存のものをそのまま使用し、振り直し・欠番の削除・範囲省略は絶対禁止。欠番(38番)もそのまま維持する。
- 完了したタスクも削除せず、取り消し線(`~~`)で全件表示し続ける。
- 表示形式は毎回「全タスク表示(完了含む・取り消し線)」で統一する。残タスクだけの絞り込み表示はしない。
- 「将来的な検討事項」セクションも毎回あわせて表示する。省略しない。
- 括弧内の詳細は一切省略せず原文通り表示する。
- 番号は行頭に`- `(ハイフン+半角スペース)を付けた箇条書きとして記載し、取り消し線はその箇条書きの中の文章全体(番号込み)にかける(例:`- ~~4. text~~`)。Markdownの自動採番付きリスト(`1.` `2.`...)は使わない。自動採番付きリストと取り消し線を組み合わせると表示側が番号を振り直してしまうバグが過去に発生したため、必ず箇条書き(`-`)の中に番号をテキストとして埋め込む形にすること。
